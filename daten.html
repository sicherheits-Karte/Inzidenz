Datenquellen & Methodik

Transparenz ist uns wichtig. Hier erfahren Sie, woher die Daten stammen, wie sie verarbeitet werden und welche Technologien wir einsetzen.

Hinweis: Dies ist keine offizielle Seite einer Behörde, sondern ein privates Projekt zur Visualisierung öffentlicher Daten.

2. Die technische Verarbeitung

Der Prozess von der Datenquelle bis zur Karte läuft vollautomatisiert ab:

Schritt 1: Datenerhebung (Scraping)

Ein Python-Skript (main.py) prüft regelmäßig die Quellen auf neue Meldungen zu Einbrüchen und Diebstählen. Relevante Texte werden extrahiert und strukturiert.

Schritt 2: Geokodierung

Damit die Vorfälle auf der Karte angezeigt werden können, müssen Ortsnamen in Koordinaten umgewandelt werden. Das Skript koordinaten.py nutzt Geodaten, um den Ort des Geschehens bestmöglich zu bestimmen.

Schritt 3: Bereitstellung

Die bereinigten Daten werden als JSON-Objekte aufbereitet und via GitHub Pages ausgeliefert. Das Frontend (die Karte) lädt diese Daten direkt in Ihren Browser.

3. Aktualisierung

Das System aktualisiert sich automatisch über GitHub Actions (CI/CD Pipeline).

  • Frequenz: Die Daten werden [z.B. täglich um 08:00 Uhr / alle 6 Stunden] aktualisiert.
  • Zeitraum: Das Dashboard zeigt standardmäßig aktuelle Daten an, lässt sich aber über den Kalender filtern.

4. Verwendete Technologien

Dieses Projekt ist Open Source und nutzt folgende Technologien:

Python GitHub Actions D3.js HTML5/CSS3 GeoJSON

5. Haftungsausschluss

Trotz sorgfältiger Verarbeitung können Fehler bei der automatischen Geokodierung oder der Datenerfassung auftreten. Die Positionen auf der Karte sind oft Näherungswerte (z.B. Ortsmitte), um keine exakten Adressen von Betroffenen preiszugeben. Alle Angaben sind ohne Gewähr.
